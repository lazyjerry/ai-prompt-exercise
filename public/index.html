02 / Compose

建立你的提示詞

01

提供相關角色、情境、受眾與現有資料，讓模型不必猜測。

02

用一句清楚的動詞開頭，說明你要模型完成什麼。

03

指定結構、長度、語言、欄位或範例。

04

列出不能假設、不能更動或不能越界的條件。

05

說明遇到哪些決策、風險或缺少資訊時，應先停下來詢問。

步驟 1 / 5

01 / Context

先說明背景是什麼

誰會使用結果？你正在處理什麼情境？模型已經有哪些資料？

02 / Request

模型到底要完成什麼

以明確動詞描述單一任務，避免把目標藏在一大段背景裡。

03 / Output Format

結果要怎麼交付

指定結構、長度、語言或必要欄位，讓結果可以直接使用。

04 / Constraints

哪些不能假設、不能越界

列出不可變更的條件、禁止事項，以及模型不應自行補完的資訊。

05 / Checkpoint

什麼時候該停下來問你

把需要你裁決的風險、範圍變更或缺少資訊寫清楚。

03 / Output

組合結果

等待輸入

完成五個欄位後，提示詞會顯示在這裡。

04 Why this works

好的提示詞，先消除模型需要猜測的地方。

Context 說明原因，Request 鎖定任務，Output Format 定義交付物，Constraints 收好邊界，Checkpoint 留下需要人類判斷的節點。
